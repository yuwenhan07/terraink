# Story Map 扩展方案文档

本目录用于沉淀 Terraink 从“静态地图海报生成器”演进到“叙事型地图作品编辑器”的设计文档。

## 文档清单

- [数据库设计](./database-design.md)
- [API 设计](./api-design.md)
- [前端目录重构方案](./frontend-restructure.md)

## 目标形态

目标产品不再只是生成一张静态地图海报，而是支持：

- 用户上传图片并自动提取 EXIF GPS
- 无 GPS 图片手动在地图上补点
- 以节点形式组织图文故事
- 在地图上点击节点查看对应图片和叙事内容
- 发布为可分享的公开作品页
- 继续复用 Terraink 的地图主题、样式生成与导出能力

## 推荐实施顺序

1. 先完成数据库模型和对象存储链路
2. 再完成上传、EXIF 提取和手动标点
3. 再完成编辑器和作品浏览页
4. 最后接回海报导出、分享图和主题模板能力

## 对现有 Terraink 的复用边界

可以直接复用：

- `MapLibre` 地图渲染
- `theme` 主题系统
- `location` 搜索和逆地理编码
- `markers` 图标与点位表达
- `export` 地图截图和导出流水线

需要新增：

- 后端服务
- 数据库和对象存储
- 作品、节点、媒体三大核心实体
- 编辑器和公开浏览页
- 发布、权限、分享链路
